# FocusFlow: 学習支援 & ポモドーロタイマーアプリケーション

## 1. プロジェクト概要

FocusFlowは、ポモドーロテクニックを用いた集中的な学習セッションを通じて、ユーザーの生産性向上を支援するために設計されたフルスタックWebアプリケーションです。ユーザーは登録、ログイン、学習タスクの管理、設定可能なポモドーロタイマーの利用、学習履歴の追跡が可能です。

**主な使用技術:**

*   **バックエンド:** Bun, Elysia.js (高速で型安全なBunのREST APIフレームワーク)
*   **フロントエンド:** React (Viteを使用), TypeScript
*   **スタイリング:** Tailwind CSS
*   **データベース:** SQLite (`bun:sqlite`経由)
*   **テスト:** `bun:test` (バックエンド), Vitest & React Testing Library (フロントエンド)

## 2. 利用開始 / セットアップ

FocusFlowアプリケーションをローカルマシンで実行するための手順です。

### 前提条件

*   **Bun:** Bunがインストールされていることを確認してください。インストール手順は [bun.sh](https://bun.sh/) で確認できます。

### インストール

1.  **リポジトリをクローン:**
    ```bash
    git clone <リポジトリURL>
    cd <リポジトリディレクトリ>
    ```

2.  **バックエンド依存関係のインストール:**
    プロジェクトのルートディレクトリから実行します:
    ```bash
    bun install
    ```
    これにより、Elysia、SQLiteドライバ、およびその他の必要なバックエンドパッケージがインストールされます。

3.  **フロントエンド依存関係のインストール:**
    `frontend`ディレクトリに移動し、依存関係をインストールします:
    ```bash
    cd frontend
    bun install
    ```
    これにより、React、Vite、Tailwind CSS、およびその他のフロントエンドパッケージがインストールされます。

### アプリケーションの実行

1.  **バックエンドサーバーの実行:**
    プロジェクトの**ルートディレクトリ**から実行します:
    ```bash
    bun run index.ts 
    ```
    (または、ルートの`package.json`に`bun run dev`のような`dev`スクリプトがある場合はそれを使用します。)
    バックエンドサーバーは通常 `http://localhost:3000` で起動します。APIもここで提供されます。

2.  **フロントエンド開発サーバーの実行:**
    **`frontend`ディレクトリ**から実行します:
    ```bash
    bun run dev
    ```
    フロントエンド開発サーバー (Vite) は通常 `http://localhost:5173` (または次に利用可能なポート) で起動します。このURLをブラウザで開いてアプリケーションを使用します。

### Docker Compose での実行

代わりに、Docker Compose を使用してアプリケーション全体を実行することもできます。

**前提条件:**

*   **Docker:** Dockerがインストールされていることを確認してください。([Dockerを入手](https://www.docker.com/get-started))
*   **Docker Compose:** Docker Composeがインストールされていることを確認してください (通常、Docker Desktopに含まれています)。

**コマンド:**

1.  **サービスのビルドと開始:**
    プロジェクトの**ルートディレクトリ** (`docker-compose.yml` がある場所) から実行します:
    ```bash
    docker-compose up --build -d
    ```
    `-d` フラグは、コンテナをデタッチモード (バックグラウンド) で実行します。

2.  **アプリケーションへのアクセス:**
    *   **フロントエンド:** ブラウザを開き、`http://localhost:8080` にアクセスします。
    *   **バックエンドAPI:** APIは `http://localhost:3001` でアクセス可能です。

3.  **ログの表示:**
    特定のサービスのログを表示するには:
    ```bash
    docker-compose logs -f backend
    ```
    またはフロントエンドの場合:
    ```bash
    docker-compose logs -f frontend
    ```
    `-f` フラグはログ出力を追跡します。

4.  **サービスの停止:**
    コンテナを停止して削除するには:
    ```bash
    docker-compose down
    ```

## 3. 機能と使い方

### ユーザー認証

認証はJSON Web Token（JWT）を使用したトークンベースで行われます。ログインまたは登録が成功すると、JWTがクライアントに発行され、クライアントは保護されたリソースへの後続のリクエストの`Authorization`ヘッダーにこのトークンを含めます。

*   **登録:**
    1.  アプリケーションにアクセスします。明示的な「登録」または「サインアップ」のリンク/ページがあればそれを使用します。(注意: 現在のUIはログイン後のタイマーに主眼を置いているため、UIに登録/ログインフォームが存在しない場合、初期セットアップのためにはAPIクライアント経由でアクセスするか、追加する必要があるかもしれません)。
    2.  API経由で登録する場合 (例: PostmanやInsomniaのようなAPIツールを使用): `/auth/register` に `POST` リクエストをJSONボディと共に送信します:
        ```json
        {
          "username": "あなたのユーザー名",
          "password": "あなたのパスワード"
        }
        ```
*   **ログイン:**
    1.  登録と同様に、UI要素が利用可能であればそれを使用します。
    2.  API経由でログインする場合: `/auth/login` に `POST` リクエストをJSONボディと共に送信します:
        ```json
        {
          "username": "あなたのユーザー名",
          "password": "あなたのパスワード"
        }
        ```
        ログインに成功すると、HTTP-onlyのセッションクッキーが設定されます。
*   **ログアウト:**
    1.  UIに「ログアウト」ボタンがあれば、それをクリックします。
    2.  API経由でログアウトする場合: `/auth/logout` に `POST` リクエストを送信します。これによりセッションがクリアされます。

### ポモドーロタイマー

ポモドーロタイマーは、作業を集中的な間隔に分割するのに役立つように設計されています。

*   **タイマー表示:**
    *   残り時間を `MM:SS` 形式で表示します。
    *   現在のモード (作業、短い休憩、長い休憩) をタイマー上部のタブ形式セレクタで示します。
*   **コントロール:**
    *   **開始:** 選択したモードのタイマーを開始または再開します。
    *   **一時停止:** 現在実行中のタイマーを一時停止します。
    *   **リセット:** タイマーを停止し、現在選択されているモードの全時間に時間をリセットします。「作業」サイクル中にリセットされた場合、現在の学習セグメントの終了をバックエンドに通知することもあります。
*   **モードの切り替え:**
    *   タイマー上部の「作業」、「短い休憩」、「長い休憩」タブをクリックして、手動でそのモードに切り替えます。タイマーは新しいモードの時間にリセットされます (まだ実行中でない場合)。
*   **タイマーの時間設定:**
    *   デフォルトの時間設定:
        *   **作業:** 25分
        *   **短い休憩:** 5分
        *   **長い休憩:** 15分 (通常、ポモドーロの「作業」サイクル4回の後に取ります)
    *   これらの時間は、タイマーコントロール下の「タイマー設定」セクションの入力フィールドを使用して設定できます。変更は、各モードのタイマーが実行中でない場合に適用されます。
*   **通知とサウンド:**
    *   各セッション (作業、短い休憩、長い休憩) の終了時に、ブラウザに通知が表示されます。
    *   サウンドアラートも再生されます。このサイトの通知許可をブラウザに与える必要がある場合があります。

### 学習セッションの記録

この機能は、何に取り組んでいるかと、完了したポモドーロサイクルの数を追跡するのに役立ちます。

*   **タスクの入力:**
    *   「学習セッション」セクションで、タスクの説明を入力フィールドに入力します (例: 「歴史の教科書の第5章に取り組む」)。
*   **自動セッション開始:**
    *   「学習セッション」セクションにタスクの説明を入力した場合、ポモドーロタイマーで「作業」サイクルを開始すると、バックエンドで新しい学習セッションが自動的に作成され、開始されます。
    *   タスクが入力されていない場合は、最初に「学習セッション」セクションの「新しいセッションを開始」をクリックする必要があるかもしれません。
*   **ポモドーロカウントの更新:**
    *   ポモドーロタイマーで「作業」サイクルを完了するたびに、アクティブな学習セッションの「完了したポモドーロ」数が自動的に増加します。
*   **手動セッション終了:**
    *   「学習セッション」セクションの「現在のセッションを終了」ボタンをクリックして、現在のタスクの追跡を手動で停止します。これにより、終了時刻と合計時間が記録されます。
    *   「作業」サイクル中にポモドーロタイマーをリセットすることも、現在の学習セグメントを終了し、バックエンドセッションを更新します。
*   **記録される情報:**
    *   各学習セッションには、ユーザーID、タスクの説明、開始時刻、終了時刻、合計時間 (分単位)、完了したポモドーロサイクル数が記録されます。

### 学習履歴

*   **履歴の表示:**
    *   「学習履歴」セクション (通常、タイマー/ロガーの横または下に表示されます) に、過去の学習セッションが一覧表示されます。
    *   最新のセッションが最初に表示される時系列リストです。
*   **表示される情報:**
    *   **タスクの説明:** 取り組んだ内容。
    *   **開始時刻:** セッションが開始された日時。
    *   **終了時刻:** セッションが終了した日時 (または進行中の場合は「進行中」)。
    *   **合計時間:** セッションに費やした合計時間。
    *   **ポモドーロ数:** 完了した作業サイクル数。

### ダークモード

*   アプリケーションは、オペレーティングシステムのライトモードまたはダークモードの配色設定を尊重します。
*   ブラウザの開発者ツールを使用してページの`<html>`要素に`dark`クラスを追加することで、手動でダークモードを有効にすることもできます (これはTailwind CSSのダークモードがJavaScript経由で切り替えられる一般的な方法ですが、このアプリではUI切り替えボタンは明示的に説明されていません)。

## 4. テストの実行

自動テストを実行するためのコマンド:

*   **バックエンドテスト:**
    プロジェクトの**ルートディレクトリ**から実行します:
    ```bash
    NODE_ENV=test bun test
    ```
    これにより、Elysia APIエンドポイント (認証、セッション) のテストが実行されます。

*   **フロントエンドテスト:**
    **`frontend`ディレクトリ**から実行します:
    ```bash
    bun test 
    ```
    (または、`frontend/package.json`で指定されていれば `bun run test`)
    これにより、VitestとReact Testing Libraryを使用したコンポーネントテストが実行されます。
    *注意: 以前の自動化ワーカーによるフロントエンドテスト依存関係のインストール試行は、環境の問題に直面しました。これらの依存関係 (`vitest`, `@testing-library/react`など) がインストールされていない場合、これらのテストは実行できません。問題が発生した場合は、`cd frontend && bun install` を介して適切にインストールされていることを確認してください。*

## 5. プロジェクト構成

主要なディレクトリの簡単な概要:

*   **`/` (ルートディレクトリ):**
    *   バックエンドElysiaアプリケーションのセットアップ (`index.ts`, `db.ts`) を含みます。
    *   `db.sqlite`: SQLiteデータベースファイル (テスト中は`test_db.sqlite`)。
    *   `auth.test.ts`, `sessions.test.ts`: バックエンドテストファイル。
    *   `package.json`, `bun.lockb`: バックエンドの依存関係とスクリプト。
    *   `README.md`, `README.en.md`: プロジェクトドキュメント。
*   **`/frontend`:** Reactフロントエンドアプリケーションを含みます。
    *   `package.json`, `bun.lockb`: フロントエンドの依存関係とスクリプト。
    *   `vite.config.ts`: ViteおよびVitestの設定。
    *   **`/frontend/src`**: Reactアプリのソースコード。
        *   `main.tsx`: Reactアプリのメインエントリポイント。
        *   `App.tsx`: UIを調整するルートReactコンポーネント。
        *   **`/frontend/src/components`**: 再利用可能なReactコンポーネント (例: `PomodoroTimer.tsx`, `StudyLogger.tsx`, `StudyHistory.tsx`)。
        *   **`/frontend/src/services`**: バックエンドAPIと対話するためのモジュール (例: `sessionApi.ts`)。
        *   `index.css`: グローバルスタイルとTailwind CSSディレクティブ。
        *   `setupTests.ts`: Vitestセットアップファイル。
